01🎯 Contexto

El equipo SMC opera 23 agentes Claude · ~80 memorias · gobernanza con 31 docs · 17 connectors. Esta complejidad vive en archivos .md y .json locales · accesibles solo via CLI. Para Sebas (sin background técnico profundo) · navegar este sistema en terminal es barrera alta.

💡 Insight: La intranet es la "ventana visual" al SMC OS · GUI sobre archivos. Permite onboarding rápido + visibilidad ejecutiva.

02🔥 Problema

  • Sin GUI · todo el sistema vive en archivos invisibles para no-developers
  • Gobernanza compleja (31 docs) sin search ni navegación visual
  • MCPs disponibles sin catálogo visual · usuario no sabe qué puede hacer
  • Métricas proyecto dispersas · sin dashboard ejecutivo
  • Sebas Windows no puede usar CLI tan fluido como Mac

03💡 Hipótesis

Hipótesis: Dashboard Next.js + Supabase + 19 secciones modulares = onboarding Sebas en <1 día · visibilidad ejecutiva Guillermo · 0 CLI requerida para operación diaria.
