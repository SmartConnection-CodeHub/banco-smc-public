Pivot Estratégico Comercial

Decisión clave: Salir del posicionamiento SAP-céntrico y apuntar directamente al ecosistema PYME chileno: Bsale (+8,000 clientes) + Jumpseller (+15,000 tiendas). Mercado 10x más grande, ciclo de venta 5x más corto, y SMC ya tiene integraciones en producción.
⛔ Antes (posicionamiento SAP)
SAP-Céntrico
  • Ciclo de venta largo (90-180 días)
  • Requiere consultora SAP certificada
  • Deal size alto pero pocas oportunidades
  • Competencia de Big 4 y SI globales
  • Barrera de entrada: certificaciones SAP
  • Mercado limitado: ~200 empresas Chile
  • ICPs no ajustados al mercado real de SMC
→
✅ Ahora (posicionamiento PYME)
PYME Bsale/Jumpseller
  • Ciclo de venta corto (14-30 días)
  • SMC ya tiene integraciones en producción
  • Volumen: 23,000+ empresas potenciales
  • Competencia fragmentada, sin líderes claros
  • Entry point: Discovery Sprint 15-40 UF
  • Mercado activo: Bsale 8K + Jumpseller 15K
  • ICPs específicos: Gerente TI + Dueño PYME

Próximos Pasos

  1. 1
    Activar Domain-Wide Delegation (DWD) en Google Admin
    Ir a admin.google.com → Seguridad → Controles de API → Domain-Wide Delegation. Agregar Client ID de la Service Account con scope https://www.googleapis.com/auth/drive. Desbloquea el endpoint POST /api/docs/sync completamente.
    Pipeline ⏱ 15 min P2 — Alta
  2. 2
    Crear 4 ICPs PYME para mercado Bsale + Jumpseller
    ICP-BSALE-GERENTETI.md, ICP-BSALE-DUENOPYME.md, ICP-JUMPSELLER-GERENTETI.md, ICP-JUMPSELLER-DUENOPYME.md. Cada uno con: demografía, firmografía, pain points Bsale/Jumpseller específicos, jobs-to-be-done, señales de compra y tácticas de outreach.
    Panchita Comercial ⏱ 2h P2 — Alta
  3. 3
    Maqueta nueva landing page con posicionamiento PYME
    Rediseñar smconnection.cl para reflejar el nuevo ICP. Hero section: "Automatiza tu Bsale y Jumpseller". Testimonios de PYME. Pricing en UF con modelos Discovery + SaaS Start + Build Premium. CTA principal: agendar Discovery Sprint. Benchmark 3-5 competidores del rubro.
    Panchita ⏱ 4h P2 — Alta
  4. 4
    Deploy docs estratégicos a Google Drive (workaround Puppeteer)
    Mientras DWD no está activo: usar script Puppeteer con Chrome autenticado para subir los 8 documentos de ~/Claude/DocumentacionSMC/ a la carpeta /comercial en Drive.
    Pipeline ⏱ 30 min P3 — Media
  5. 5
    Implementar nueva landing y hacer push a S3/CloudFront
    Una vez aprobada la maqueta de Panchita: Fiori implementa en HTML puro estático. Pipeline deploya via scripts/deploy-s3.sh → S3 sync → CloudFront invalidation. No --delete, no Astro, HTML puro.
    Fiori Pipeline ⏱ 2h P2 — Alta
