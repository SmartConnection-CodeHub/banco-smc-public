Misiones — Learning path desde cero

8 misiones ordenadas de principiante a avanzado. Cada misión usa las herramientas de tu stack smconnection con objetivos concretos y pasos claros.

01
MISIÓN 01 · PRINCIPIANTE · Semana 1

Dominar el prompting con Claude y Groq

Aprender a escribir prompts que obtengan resultados consistentes. La base de todo lo que construirás después en smconnection.

  • 01.Leer el Prompt Engineering Guide de Anthropic (2h). Aprender system prompts, few-shot, chain-of-thought.
  • 02.En el console.anthropic.com, reescribir los prompts del Generador de Propuestas SAP actual. Compara resultados.
  • 03.Repetir en Groq Console con llama-3.3-70b. Observar diferencias de velocidad y calidad.
  • 04.Crear una librería de prompts reutilizables en Airtable con su resultado esperado.
Claude ConsoleGroq ConsoleAirtable
02
MISIÓN 02 · PRINCIPIANTE · Semana 2

Tu primera app Next.js con IA integrada

Construir un mini-chat usando Next.js + Vercel AI SDK conectado a Claude. La base del patrón que usarás en smconnection.

  • 01.Crear proyecto Next.js: npx create-next-app@latest --typescript --tailwind
  • 02.Instalar Vercel AI SDK: npm install ai @anthropic-ai/sdk
  • 03.Crear una API route /api/chat con streaming usando el hook useChat.
  • 04.Conectar tu API key de Claude desde la intranet smconnection.
  • 05.Deploy en Vercel o AWS Amplify. Probar en producción.
Next.jsVercel AI SDKClaude APITypeScriptTailwind
03
MISIÓN 03 · INTERMEDIO · Semana 3

Conectar Supabase + pgvector para RAG

Crear tu primer sistema RAG sobre los documentos de smconnection. Indexar, buscar semánticamente, responder con fuentes.

  • 01.En Supabase, habilitar la extensión pgvector con: CREATE EXTENSION vector;
  • 02.Crear tabla documents con columna embedding vector(1536) para OpenAI embeddings.
  • 03.Script de indexado: leer PDFs de propuestas SAP → OpenAI text-embedding-3-small → guardar en Supabase.
  • 04.Función de búsqueda semántica: match_documents con cosine similarity.
  • 05.Conectar con Claude para que responda usando los documentos encontrados como contexto.
SupabasepgvectorOpenAI EmbeddingsClaudeNext.js
04
MISIÓN 04 · INTERMEDIO · Semana 4

Tool Use: Claude llama APIs externas

Hacer que Claude llame a la API de Bsale o Jumpseller para responder preguntas sobre stock, ventas o clientes en lenguaje natural.

  • 01.Estudiar Tool Use en los docs de Anthropic (2h). Entender el patrón define_tool → call → result → response.
  • 02.Definir herramientas JSON para Bsale: get_products, get_invoices, get_clients.
  • 03.Implementar el handler que cuando Claude llama la herramienta, ejecuta la API de Bsale/Jumpseller real.
  • 04.Probar con: "¿Cuántas facturas emitimos en marzo?" y validar que Claude obtiene el dato real.
Claude Tool UseBsale APIJumpseller APINext.js API routes
05
MISIÓN 05 · INTERMEDIO · Semana 5

Hoku: agregar un modelo nuevo + observabilidad

Integrar un modelo adicional en Hoku (ej: DeepSeek R1 o Claude Opus) y agregar Langfuse para trazar qué modelo responde qué.

  • 01.Registrarse en cloud.langfuse.com. Instalar el SDK: npm install langfuse
  • 02.Instrumentar todas las llamadas de Hoku con langfuse.trace() para registrar model, tokens, latencia, costo.
  • 03.Añadir DeepSeek R1 vía OpenRouter (compatible con OpenAI SDK) como décimo primer modelo.
  • 04.En el dashboard de Langfuse, analizar: ¿qué modelo es más lento? ¿cuál alucina más?
HokuLangfuseOpenRouterDeepSeek R1TypeScript

MISIÓN 06 · AVANZADO · Semana 6

Crear un MCP Server para smconnection

Construir un MCP Server personalizado que exponga las APIs de smconnection (Supabase, Bsale, Airtable) a Claude de forma nativa.

  • 01.Leer la spec de MCP en modelcontextprotocol.io y el quickstart de Anthropic (3h).
  • 02.Crear el proyecto MCP: npx @modelcontextprotocol/create-server smconnection-mcp
  • 03.Implementar tools: query_supabase, get_bsale_report, list_airtable_records, send_resend_email.
  • 04.Configurar el MCP en Claude Desktop y en la intranet para que Hoku lo use automáticamente.
  • 05.Publicar en el MCP Registry para que el equipo lo use.
MCP SDKClaude MCPSupabaseBsale APIResend

MISIÓN 07 · AVANZADO · Semana 7

GitHub Actions + DeepSeek: DevSecOps pipeline

Automatizar code review de seguridad en cada PR usando GitHub Actions + DeepSeek-Coder. Solo alerta si hay vulnerabilidades críticas.

  • 01.Crear .github/workflows/ai-review.yml con trigger on pull_request.
  • 02.El Action extrae el diff del PR y lo envía a DeepSeek-Coder vía API para análisis de seguridad.
  • 03.Si DeepSeek detecta vulnerabilidades (SQL injection, secrets expuestos, XSS), el Action falla y notifica.
  • 04.Integrar Resend para enviar el reporte de seguridad al email del desarrollador.
  • 05.Guardar historial de revisiones en Supabase para tracking.
GitHub ActionsDeepSeek-CoderResendSupabaseYAML

MISIÓN 08 · AVANZADO · Semana 8 — PROYECTO FINAL

InfoPet: asistente veterinario completo con RAG

Construir el asistente IA de InfoPet end-to-end: literatura veterinaria indexada, diagnóstico asistido, historial de mascotas y recordatorios automáticos.

  • 01.Indexar literatura veterinaria en Supabase pgvector usando OpenAI Embeddings o Cohere Embed v3.
  • 02.Implementar el chat veterinario con Claude (safety mode) + RAG sobre los documentos indexados.
  • 03.Crear tabla mascotas en Supabase: nombre, especie, fechas de vacunas, historial de consultas.
  • 04.Conectar Bsale para facturar consultas y Jumpseller para vender productos desde la app.
  • 05.Configurar recordatorios automáticos de vacunas via Resend + Google Calendar.
  • 06.Deploy en AWS Amplify con CI/CD via GitHub Actions. Dominio custom en Route 53.
