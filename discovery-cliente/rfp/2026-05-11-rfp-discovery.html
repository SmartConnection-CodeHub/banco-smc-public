01🎯 Contexto

Business-Dev SMC tiene reuniones con prospectos semanales · cada una genera transcript grabado (Otter · Zoom) que después queda en limbo. Análisis SPIN/BANT manual toma 45-60 min · propuesta cotizable otros 30-45 min. Total 90-120 min por prospecto. Si Business-Dev tiene 10 reuniones/semana · son 15-20 horas en post-procesamiento.

💡 Insight: El transcript no es el problema · es el análisis estructurado SPIN/BANT y la propuesta. Con Claude API + plantillas + Discovery app · reducimos a 5 min con calidad superior.

02🔥 Problema

  • Transcripts quedan en limbo · sin análisis SPIN/BANT
  • Business-Dev gasta 90-120 min/prospecto en post-procesamiento
  • Calidad inconsistente entre análisis (depende del operario)
  • Propuestas tardan días en llegar al prospecto · pierde momentum
  • Sin trazabilidad histórica de qué se cerró · qué se perdió · por qué

03💡 Hipótesis

Hipótesis: Pipeline transcript → SPIN/BANT → infografía + reporte + propuesta automatizado en 3 fases (manual con plantillas → script bash → app web self-service) reduce post-procesamiento de 90 min a 5 min · libera 15-20 horas/semana de Business-Dev.

04🛠️ Solución · 3 fases

Fase 1 · Manual con plantillas (✅ ACTIVO)

  • Plantillas HTML editables para infografía · reporte · propuesta
  • Business-Dev copia transcript · pega · análisis SPIN/BANT manual con guía
  • Tiempo: ~30 min por discovery

Fase 2 · Script bash + Claude (🔄 WIP)

  • Script lee transcript .txt · llama Claude API con prompt SPIN/BANT versionado
  • Genera HTMLs automáticos (infografía + reporte + propuesta)
  • Tiempo: ~5 min por discovery

Fase 3 · App web self-service (⏳ FUTURO)

  • Cliente sube transcript desde intranet.smconnection.cl
  • Sistema genera todo · cliente descarga PDF · email automático
  • Tiempo: ~1 min cliente · cero operario SMC
