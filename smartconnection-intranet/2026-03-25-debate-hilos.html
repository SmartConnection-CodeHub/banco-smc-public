Oportunidad: Ninguna plataforma combina debates entre agentes IA con sistema de hilos, orquestacion en tiempo real, deteccion de tensiones y perspectivas temporales. SmartConnection seria la primera intranet con debate multi-agente controlable + mapa de tensiones para decisiones humanas.

2. User Stories

US-1: Iniciar debate multi-agente

"Como Guillermo, quiero lanzar un tema para que multiples agentes IA debatan entre ellos, para obtener perspectivas diversas sin consultar uno por uno."

  • DADO que estoy en /dashboard/agents y selecciono modo "Debate"
  • CUANDO escribo "Mejor stack para landing de conversion en Chile 2026" y selecciono Hoku, Groq, Claude, DeepSeek
  • ENTONCES los 4 agentes responden secuencialmente en el chat grupal, cada uno con su color e icono, respondiendo al tema y reaccionando a lo que dijeron los anteriores

Edge cases: Un agente falla → se muestra error inline y el debate continua con los demas. Timeout de 30s por agente.

US-2: Ver debate en tiempo real con streaming

"Como Guillermo, quiero ver las respuestas de los agentes aparecer en streaming, para no esperar a que todos terminen."

  • DADO que un debate esta en curso con 4 agentes
  • CUANDO el primer agente comienza a responder
  • ENTONCES veo el texto aparecer token por token con indicador de "escribiendo", y cuando termina, el siguiente agente comienza automaticamente

US-3: Abrir hilo desde un mensaje

"Como Guillermo, quiero abrir un hilo lateral desde cualquier mensaje del debate, para profundizar en un punto especifico sin interrumpir la conversacion principal."

  • DADO que un agente dijo "Astro es superior para SEO estatico"
  • CUANDO hago clic en el icono de hilo del mensaje
  • ENTONCES se abre un panel lateral derecho con el mensaje citado, un campo para escribir, y puedo pedir a cualquier agente que profundice

Edge cases: Multiples hilos abiertos → pestanas para navegar. Hilo sin respuestas → se muestra estado vacio con CTA.

US-4: Aprobar o rechazar propuestas en hilos

"Como Guillermo, quiero aprobar o rechazar propuestas que surjan en los hilos, para tomar decisiones ejecutivas sobre lo que se implementa."

  • DADO que en un hilo, Claude propuso migrar el hero a Astro con View Transitions
  • CUANDO hago clic en "Aprobar"
  • ENTONCES el hilo se marca como "Aprobado", se registra en Supabase, y opcionalmente se genera una tarea para el agente correspondiente

US-5: Continuar debate con intervencion humana

"Como Guillermo, quiero intervenir en medio de un debate para redirigir la discusion, para que los agentes no se desvien del tema."

  • DADO que un debate sobre "estrategia de pricing" se desvio a temas tecnicos
  • CUANDO escribo "Enfoquense en el pricing de competidores chilenos, no en stack"
  • ENTONCES mi mensaje aparece destacado como intervencion del usuario, y los agentes ajustan sus proximas respuestas al nuevo enfoque

US-6: Director de Orquesta — controlar participantes por ronda SERGITO

"Como Guillermo, quiero controlar que agentes participan en cada ronda del debate, para enfocar la discusion entre voces especificas."

  • DADO que un debate esta en curso con 4 agentes
  • CUANDO selecciono modo "Dueto" y activo solo Hoku y Claude
  • ENTONCES la siguiente ronda solo incluye a Hoku y Claude, los demas aparecen desactivados (pero pueden reactivarse en cualquier momento)

Modos disponibles:

  • Tutti — todos los agentes seleccionados debaten (modo por defecto)
  • Dueto — seleccionar exactamente 2 agentes para confrontar posiciones
  • Solo — un agente profundiza (los demas escuchan pero no responden)

Edge cases: Cambiar modo mid-debate → aplica desde la siguiente ronda. Toggle de agente individual siempre disponible. Si quedan 0 agentes activos → warning "Selecciona al menos 1 agente".

US-7: Mapa de Tensiones — visualizar desacuerdos SERGITO

"Como Guillermo, quiero ver donde los agentes no se ponen de acuerdo, para identificar los puntos de decision real."

  • DADO que Hoku dice "Astro gana por goleada" y Claude dice "Next.js ofrece mejor DX para formularios complejos"
  • CUANDO el sistema detecta posiciones contradictorias entre agentes
  • ENTONCES el mensaje se marca con badge "Tension" naranja, y al final del debate aparece un resumen de puntos de tension con los agentes involucrados

Deteccion: El backend analiza semanticamente cada respuesta contra las anteriores. Si detecta contradiccion, oposicion o matiz fuerte, marca el mensaje con tension_with: ['agent_id']. Umbral configurable.

Edge cases: Debate sin tensiones → no se muestra el resumen. Tension resuelta en rondas posteriores → se marca como "resuelta". Click en tension → scroll al mensaje conflictivo.

US-8: Perspectivas Temporales — cada agente habla desde un horizonte SERGITO

"Como Guillermo, quiero que cada agente analice desde un horizonte temporal diferente, para obtener vision completa: desde la implementacion inmediata hasta el impacto a 3 anos."

  • DADO que inicio un debate con perspectivas temporales activadas
  • CUANDO los agentes responden
  • ENTONCES cada agente responde desde su horizonte asignado, y su mensaje muestra un badge con el horizonte temporal

Horizontes por defecto (agentes IA + equipo):

AgenteHorizonteEnfoque
Hoku1 sprint despuesImplementacion inmediata, viabilidad tecnica, esfuerzo
Panchita6 meses despuesArquitectura, escalabilidad, mantenibilidad
CamilitaEl dia del incidenteQue se rompe, puntos de falla, disaster recovery
ArielitoAuditoria anualCompliance, deuda tecnica, documentacion

Edge cases: Perspectivas temporales es opcional (toggle on/off al crear debate). Si esta desactivado, los agentes debaten libremente. Agentes IA sin horizonte fijo pueden recibir uno asignado por el usuario.

4. Flujo de usuario

Flujo 1: Iniciar debate (actualizado con Director de Orquesta)

  1. Usuario hace clic en "Nuevo debate" en la barra superior
  2. Se abre modal: titulo, tema/prompt, seleccion de agentes (chips clicables)
  3. NUEVO: Selector de modo Director de Orquesta (Tutti / Dueto / Solo) con iconos
  4. NUEVO: Toggle "Perspectivas temporales" — si se activa, muestra config de horizonte por agente
  5. Minimo 2 agentes, maximo 6 para mantener coherencia (en modo Solo: minimo 1)
  6. Click "Iniciar debate" → se crea registro en debates con orchestration_mode y temporal_config
  7. Backend lanza agentes segun modo: Tutti=todos, Dueto=par, Solo=uno
  8. Streaming SSE muestra cada respuesta token por token
  9. Si temporal_enabled, cada agente recibe instruccion de horizonte en su system prompt

Flujo 2: Respuesta de agentes (backend, actualizado)

  1. Primer agente recibe: system prompt + tema del debate + horizonte temporal (si aplica)
  2. Segundo agente recibe: system prompt + tema + respuesta del agente 1 + su horizonte
  3. N-esimo agente recibe: tema + resumen de respuestas anteriores (max 2000 tokens de contexto)
  4. Cada respuesta se guarda en debate_messages con time_horizon
  5. NUEVO: Despues de cada respuesta, el backend analiza semanticamente contra respuestas previas para detectar tensiones. Si detecta contradiccion → guarda en debate_tensions y actualiza tension_with en el mensaje
  6. Al terminar todos, status del debate pasa a "completed"

Flujo 3: Director de Orquesta en tiempo real

  1. Barra de orquestacion visible debajo del header del debate
  2. Botones de modo: Tutti / Dueto / Solo (el activo se resalta)
  3. Toggle individual en cada chip de agente: click para activar/desactivar
  4. Agente desactivado aparece con opacidad reducida y texto tachado
  5. Cambio de modo aplica a la siguiente ronda — no interrumpe la ronda en curso
  6. PATCH a /api/debates/:id actualiza orchestration_mode y active_agent_ids

Flujo 4: Mapa de Tensiones

  1. Mensajes con tension se marcan con borde izquierdo naranja + badge "Tension"
  2. Badge muestra con quien es la tension (ej: "Tension con Hoku")
  3. Click en badge → scroll al mensaje contradictorio
  4. Al completar debate (o en cualquier momento), barra colapsable de "Mapa de tensiones" aparece arriba del input
  5. Resumen lista cada tension: agentes involucrados + descripcion breve
  6. Tensiones resueltas en rondas posteriores se marcan como "Resuelta"

Flujo 5: Abrir hilo

  1. Hover sobre mensaje → aparecen acciones: [hilo] [copiar] [citar]
  2. Click en [hilo] → se abre panel lateral derecho (420px)
  3. Panel muestra: mensaje citado arriba, respuestas debajo, input abajo
  4. Se crea registro en threads con parent_msg_id
  5. Se incrementa thread_count en el mensaje padre
  6. El mensaje en la conversacion principal muestra indicador "3 respuestas en hilo"

Flujo 6: Navegar entre hilos

  1. Multiples hilos abiertos se muestran como pestanas en la parte superior del panel
  2. Click en pestana → cambia el contenido del panel
  3. Pestana con punto verde = mensajes nuevos sin leer
  4. Click en X → cierra la pestana (no elimina el hilo)

Flujo 7: Aprobar/Rechazar

  1. En cada hilo, botones "Aprobar" y "Rechazar" visibles
  2. Click en "Aprobar" → actualiza threads.decision = 'approved'
  3. Opcionalmente agrega nota de decision
  4. El hilo se marca visualmente como aprobado/rechazado
  5. Se puede revertir la decision

6. SQL de creacion

-- Tabla debates (actualizada)
CREATE TABLE IF NOT EXISTS debates (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title text NOT NULL,
  topic text NOT NULL,
  agent_ids text[] NOT NULL DEFAULT '{}',
  status text NOT NULL DEFAULT 'active' CHECK (status IN ('active','paused','completed','archived')),
  orchestration_mode text NOT NULL DEFAULT 'tutti' CHECK (orchestration_mode IN ('tutti','dueto','solo')),
  active_agent_ids text[] DEFAULT '{}',
  temporal_enabled boolean DEFAULT false,
  temporal_config jsonb DEFAULT '{}',
  created_by text NOT NULL DEFAULT 'guillermo',
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);

-- Tabla debate_messages (actualizada)
CREATE TABLE IF NOT EXISTS debate_messages (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  debate_id uuid NOT NULL REFERENCES debates(id) ON DELETE CASCADE,
  author_type text NOT NULL CHECK (author_type IN ('agent','user')),
  author_id text NOT NULL,
  content text NOT NULL,
  tokens_used integer DEFAULT 0,
  duration_ms integer DEFAULT 0,
  status text DEFAULT 'completed' CHECK (status IN ('streaming','completed','error')),
  metadata jsonb DEFAULT '{}',
  time_horizon text,
  tension_with text[] DEFAULT '{}',
  thread_count integer DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Tabla debate_tensions (NUEVA)
CREATE TABLE IF NOT EXISTS debate_tensions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  debate_id uuid NOT NULL REFERENCES debates(id) ON DELETE CASCADE,
  msg_a_id uuid NOT NULL REFERENCES debate_messages(id) ON DELETE CASCADE,
  msg_b_id uuid NOT NULL REFERENCES debate_messages(id) ON DELETE CASCADE,
  agent_a text NOT NULL,
  agent_b text NOT NULL,
  description text,
  resolved boolean DEFAULT false,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Tabla threads
CREATE TABLE IF NOT EXISTS threads (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  debate_id uuid NOT NULL REFERENCES debates(id) ON DELETE CASCADE,
  parent_msg_id uuid NOT NULL REFERENCES debate_messages(id) ON DELETE CASCADE,
  title text,
  status text NOT NULL DEFAULT 'open' CHECK (status IN ('open','closed')),
  decision text CHECK (decision IN ('approved','rejected')),
  decision_note text,
  created_by text NOT NULL DEFAULT 'guillermo',
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);

-- Tabla thread_messages
CREATE TABLE IF NOT EXISTS thread_messages (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  thread_id uuid NOT NULL REFERENCES threads(id) ON DELETE CASCADE,
  author_type text NOT NULL CHECK (author_type IN ('agent','user')),
  author_id text NOT NULL,
  content text NOT NULL,
  tokens_used integer DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Indices
CREATE INDEX idx_debate_messages_debate ON debate_messages(debate_id, created_at);
CREATE INDEX idx_threads_debate ON threads(debate_id);
CREATE INDEX idx_threads_parent ON threads(parent_msg_id);
CREATE INDEX idx_thread_messages_thread ON thread_messages(thread_id, created_at);
CREATE INDEX idx_debates_status ON debates(status, created_at DESC);
CREATE INDEX idx_debate_tensions_debate ON debate_tensions(debate_id);
CREATE INDEX idx_debate_messages_tension ON debate_messages(debate_id) WHERE tension_with != '{}';

-- Trigger updated_at
CREATE OR REPLACE FUNCTION update_updated_at() RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER debates_updated_at BEFORE UPDATE ON debates
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER threads_updated_at BEFORE UPDATE ON threads
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
