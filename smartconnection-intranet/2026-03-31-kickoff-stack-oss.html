Arielito — Decisión arquitectónica
Arquitectura dual-CLI diseñada y documentada
Problema resuelto: ¿qué hacer cuando se acaban los créditos de Claude Code sin perder contexto ni workflow?

Decisión final (ADR): NO routing transparente. Dos CLIs explícitas que comparten el mismo contexto via git:
 • Claude Code → sesiones con créditos, tareas complejas, agentes completos
 • Aider CLI → sesiones sin créditos, conectado a Ollama local (gratis, sin límites)

Opciones descartadas:
 • Antigravity (Electron) → no CLI, pesado, no añade valor real
 • Routing transparente automático → demasiado frágil, contexto diferente por CLI
 • OpenRouter directo en Claude Code → gasta créditos igual

Branch strategy:
 • main → Claude Code únicamente (producción)
 • dev → Aider por defecto (experimental)
 • feature/* → cualquiera de las dos CLIs

Regla de sincronización: al cerrar sesión → sync.sh push. Al abrir → sync.sh pull. Ambas CLIs trabajan con el mismo ~/claude-config/ local.

Hoku — Research de herramientas OSS
Ecosistema de herramientas existentes evaluado
Aider CLI ADOPTAR Fase 1
 Open source coding CLI. Lee todo el codebase, edita archivos, hace git commits. Funciona con Ollama local (gratis) o cualquier API (OpenRouter). Más maduro que alternativas (Continue, Cursor local).

Engram MCP ADOPTAR Fase 4
 "Cerebro compartido" persistente entre sesiones y entre CLIs via git. Almacena contexto, decisiones, estado de proyectos. Tanto Claude Code como Aider pueden leer/escribir el mismo brain.

LiteLLM Proxy ADOPTAR Fase 3
 Router de modelos unificado. Un endpoint local que enruta: simple→Ollama (gratis), medio→Groq (gratis con rate limit), complejo→Claude vía OpenRouter (key ya en Amplify). Permite cambiar de modelo sin tocar código.

Langfuse ADOPTAR Fase 5
 LLMOps: logs de cada llamada IA, costo por feature, latencia p50/p95, tasa de error. Self-hosted gratis. Funciona con Claude Code, Aider y cualquier provider vía LiteLLM.

OpenRouter DISPONIBLE
 300+ modelos vía API unificada. Key ya configurada en Amplify. Permite usar Llama, Mistral, Gemini, Claude sin cuentas separadas. Precio por token, no por suscripción.

Descartados:
 • OpenClaw / Antfarm → multi-agent scheduling, demasiado complejo para el caso de uso actual
 • Continue (VS Code) → solo extensión, no CLI independiente
 • Cursor local → cerrado, no compatible con flujo de agentes

Pipeline — Fase 1 P1 · Primer paso
Instalar Aider CLI y configurar para Ollama local
¿Qué es Aider? CLI open source de coding que lee el codebase completo, edita archivos y hace git commits automáticos. Se conecta directamente a Ollama (sin costo, sin límites).

Comandos:
pip install aider-chat
aider --version  ← verificar instalación

Configuración global → crear ~/.aider.conf.yml:
model: ollama/qwen2.5-coder:7b
auto-commits: true
git: true
dark-mode: true

Test de funcionamiento:
cd ~/infopet-next
ollama serve &  ← arrancar Ollama en background
aider src/app/page.tsx  ← Aider lee el archivo y espera instrucciones

¿Cómo se usa? En el prompt de Aider escribes en lenguaje natural: "agrega un botón de logout al header" → Aider edita los archivos y hace el commit. Sin tokens de Claude Code.

Pipeline — Fase 3 P2
LiteLLM Proxy — router inteligente de modelos
¿Qué resuelve? Un solo endpoint local (localhost:4000) que enruta automáticamente al modelo correcto según la complejidad. Tanto Claude Code como Aider apuntan a este proxy.

Instalación:
pip install 'litellm[proxy]'

Crear ~/litellm-config.yaml:
model_list:
  - model_name: fast
    litellm_params:
      model: ollama/qwen2.5-coder:7b  ← gratis, local
  - model_name: medium
    litellm_params:
      model: groq/llama-3.3-70b-versatile  ← gratis rate limit
  - model_name: advanced
    litellm_params:
      model: openrouter/anthropic/claude-sonnet-4-6  ← key en Amplify

Arrancar:
litellm --config ~/litellm-config.yaml --port 4000

Usar en Aider:
aider --openai-api-base http://localhost:4000 --model fast

Hoku — Fase 4 P2
Engram MCP — memoria persistente y compartida entre CLIs
¿Qué resuelve? Ambas CLIs (Claude Code y Aider) pueden leer y escribir en el mismo "cerebro" — decisiones tomadas, contexto de proyectos, estado de tareas. Se persiste en git, no en memoria RAM.

Instalación:
npm install -g @empathize/engram-mcp

Agregar a ~/.claude/settings.json en la sección mcpServers:
"engram": {
  "command": "engram-mcp",
  "args": ["--brain-dir", "/Users/guillermogonzalezleon/engram-brain"]
}

Inicializar brain repo:
mkdir ~/engram-brain && cd ~/engram-brain
git init && git remote add origin git@github.com:guillermogonzalezleon-lgtm/engram-brain.git

Resultado: Claude Code puede guardar contexto con engram save "..." y Aider puede recuperarlo en la próxima sesión.

Hoku — Fase 5 P2
Herramientas complementarias del stack OSS
Langfuse — observabilidad LLM (costos, tokens, latencia por feature):
pip install langfuse
Self-hosted gratis. Dashboard con: costo por llamada, latencia p50/p95, prompts usados, errores. Conectar via LiteLLM callbacks.

Tavily — búsqueda web para Aider (cuando necesite buscar docs):
pip install tavily-python
API key gratuita con 1000 búsquedas/mes. Aider puede buscar documentación actualizada sin depender del contexto de entrenamiento.

direnv — variables de entorno por directorio:
brew install direnv
Crea .envrc en cada proyecto → al entrar al directorio se cargan automáticamente las env vars. Útil para API keys de Ollama, LiteLLM, Langfuse.

fswatch — watcher de archivos para auto-sync memorias:
brew install fswatch
Detecta cambios en ~/.claude/projects/.../memory/ → dispara sync.sh push automáticamente sin depender del hook Stop.

repomix — empaquetar codebase para contexto de Aider:
pip install repomix
Genera un único archivo con todo el código del proyecto (respetando .gitignore). Útil para darle contexto completo a Aider en proyectos grandes sin consumir RAM con cada archivo individual.
