// 05 — CASOS DE USO

Ejemplos reales de arquitectura

Escenarios concretos donde estas herramientas resuelven problemas reales. Con stack recomendado y justificación técnica.

CASO 01 — ENTERPRISE
Asistente interno de base de conocimiento
Una empresa quiere que sus empleados hagan preguntas en lenguaje natural sobre documentos internos (POCs, policies, reportes). Respuestas con fuentes citadas, sin alucinaciones.
Claude Sonnet Supabase (pgvector) Cohere Embed v3 Cohere Reranker Next.js + Vercel AI SDK
¿Por qué? Claude para razonamiento y respuesta fiel. Cohere Embed para indexar los documentos. pgvector en Supabase para búsqueda eficiente. Reranker para mejorar relevancia. Vercel AI SDK para streaming en UI.
CASO 02 — DEVTOOLS
Code review automatizado en PR
En el pipeline CI/CD, cada Pull Request recibe automáticamente un análisis de código: bugs potenciales, violaciones de estilo, sugerencias de mejora.
DeepSeek-Coder n8n (GitHub trigger) Resend (notificación) Supabase (historial)
¿Por qué? DeepSeek-Coder supera a otros modelos en tareas de análisis de código (HumanEval). Bajo costo para procesamiento masivo de PRs. n8n como orquestador del trigger. Resend para notificar al desarrollador.
CASO 03 — CUSTOMER
Chatbot de atención al cliente 24/7
Chatbot que responde consultas de clientes, escala a humano cuando es necesario, mantiene memoria de conversación y tiene baja latencia para buena UX.
Groq (Llama 3) OpenRouter (fallback Claude) Supabase (memoria) Vercel AI SDK
¿Por qué? Groq para respuestas sub-100ms. OpenRouter como fallback a Claude para casos complejos. Supabase para historial de conversación. Escalado automático con Vercel Edge.
CASO 04 — ANALYTICS
Pipeline de análisis de datos con IA
Automatizar análisis semanal de métricas de negocio: extraer datos, generar insights, crear reportes narrativos y enviarlos al equipo directivo.
n8n (scheduler) Claude Opus Airtable (datos) Resend (reporte)
¿Por qué? n8n para orquestar el flujo semanal. Claude Opus para análisis narrativo de alta calidad. Airtable como fuente de datos accesible. Resend para entregar el reporte formateado.
CASO 05 — COMPLIANCE
Clasificación y redacción legal GDPR
Empresa europea necesita clasificar documentos sensibles, redactar PIA (Privacy Impact Assessments) y asegurar que los datos nunca salgan de Europa.
Mistral Large (EU) Supabase (EU region) n8n (self-hosted)
¿Por qué? Mistral con servidores en Europa = GDPR por diseño. Supabase en región EU. n8n self-hosted para que los datos nunca salgan del tenant. Stack 100% soberano.
CASO 06 — AGENT
Agente IA autónomo con herramientas
Agente que puede buscar en la web, ejecutar código, consultar APIs externas y producir un resultado complejo (ej: investigación de mercado automatizada).
Claude Sonnet (MCP) n8n (herramientas) OpenRouter (multi-LLM) Supabase (estado)
¿Por qué? Claude con MCP para llamar herramientas de forma confiable. n8n expone las herramientas como webhooks. OpenRouter como fallback. Supabase para persistir el estado del agente entre pasos.
CASO 07 — CONTENT
Generación de contenido multicanal
Plataforma que genera artículos, posts para redes sociales, newsletters y descripciones de producto a partir de un brief inicial, en múltiples idiomas.
Claude Sonnet DeepSeek (idiomas asiáticos) OpenRouter (routing) Airtable (CMS) n8n (publish flow)
¿Por qué? Claude para inglés/español. DeepSeek para chino/japonés. OpenRouter enruta inteligentemente por idioma y costo. Airtable como CMS visual. n8n publica automáticamente en cada canal.
CASO 08 — INFOPET
Asistente veterinario IA (InfoPet)
Para tu proyecto InfoPet: diagnóstico preliminar de síntomas de mascotas, recomendaciones de cuidado, recordatorios de vacunas y conexión con veterinarios.
Claude Sonnet Supabase (historial mascota) Cohere Embed (RAG veterinario) Vercel AI SDK Resend (recordatorios)
¿Por qué? Claude por su safety en contexto médico/veterinario y capacidad de razonamiento. RAG con literatura veterinaria para respuestas fundamentadas. Supabase para historial de cada mascota. Resend para recordatorios automáticos de vacunas.
