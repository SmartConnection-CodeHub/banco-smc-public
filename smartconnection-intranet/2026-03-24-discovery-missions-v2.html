Caso de Uso

Quieres usar IA para desarrollo pero tus prompts generan respuestas genéricas. Esta misión te enseña a escribir prompts que producen código funcional, propuestas SAP, y análisis UX de alta calidad.

El Desafío

Los prompts vagos producen resultados vagos. "Hazme un login" vs un prompt estructurado con contexto, formato y restricciones produce resultados radicalmente diferentes.

La Solución

Dominar las 4 partes de un prompt efectivo y aprender técnicas avanzadas como Chain-of-thought y system prompts para diferentes modos.

Las 4 partes de un prompt

Un prompt efectivo tiene 4 componentes. Sin ellos, el modelo adivina. Con ellos, el modelo ejecuta exactamente lo que necesitas.

1. Contexto → 2. Tarea → 3. Formato → 4. Restricciones

1. Contexto

Quién eres, qué proyecto es, qué stack usas. Sin contexto, Claude no sabe si eres junior o senior, si usas React o Vue.

2. Tarea

Qué necesitas exactamente. "Hazme un login" ≠ "Crea un LoginForm.tsx con Supabase Auth, validación zod, dark mode".

3. Formato

Cómo quieres la respuesta: código con filename, bullets, tabla comparativa, JSON, etc.

4. Restricciones

Qué NO hacer: "No uses server actions", "Solo client components", "TypeScript estricto".

💡 Tip: En la intranet, los modos (Chat/Code/SAP/Deploy) ya incluyen contexto automáticamente. Solo necesitas la tarea y restricciones.
🎯 Ejercicio: Abre el chat de Hoku (🐾 abajo a la derecha). Escribe un prompt SIN contexto: "Hazme un componente". Luego uno CON las 4 partes. Compara.
